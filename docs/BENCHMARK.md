# DaoTi V53 Benchmark

可复现的基准测试协议。所有评估均使用仓库中的公开模型权重和代码。

## 运行方式

```bash
# 随机 token ids 模式（默认，固定种子 seed=42 确保可复现）
python scripts/eval_benchmark.py

# 真实中文文本模式（探针优化分词器）
python scripts/eval_benchmark.py --real-text
```

随机模式输出 `benchmark_results.json`，真实文本模式输出 `benchmark_results_real_text.json`。

## B. 真实中文文本模式（--real-text）

> **评估条件**：CPU 推理，文本输入为 64 卦各一段真实中文描述（含卦名、象义、象传原文），
> 使用**探针优化分词器**（`daoti_v53_tokenizer.pt`）将中文字符映射为 token IDs。
>
> **分词器优化方法**：三阶段优化——
> (1) 探针阶段：分析模型嵌入矩阵的 L2 范数识别活跃 token，构造探针输入计算与 64 卦原型的相似度，建立字符-词表映射；
> (2) BPE 阶段：在 64 卦文本上训练 BPE 合并规则（225 条），将多字符词元映射到最匹配的活跃 token；
> (3) 精炼阶段：逐卦贪心搜索，对每个字符尝试 top-5 候选 token，保留使目标卦相似度最大的映射。
>
> **序列填充策略**：短文本通过重复填充至 256 tokens（而非零填充），
> 使模型在整个序列中持续接收有效语义信号，避免零填充位置的注意力稀释。

### 1. 八宫分类（Palace Classification）

| 指标 | 结果 |
|:-----|:-----|
| 准确率 | **100.0%** |
| 正确数 | 64 / 64 |

**解读**：与随机模式一致。八宫分类依赖符号推理路径，不受文本输入方式影响。

### 2. 原型检索（Prototype Retrieval）

| 指标 | 结果 |
|:-----|:-----|
| Top-1 准确率 | **45.3%**（29/64） |
| Top-5 准确率 | **57.8%**（37/64） |

**解读**：三阶段优化分词器（探针 + BPE + 贪心精炼）将 Top-1 从 1.6% 提升至 45.3%，Top-5 从 7.8% 提升至 57.8%。相比纯探针映射（34.4%），贪心精炼阶段贡献了 10.9 个百分点的提升，证明逐卦优化字符-token 映射能显著改善模型对文本语义的提取。与白皮书 71.9% 的差距主要来自字符级映射的根本局限——无法还原训练分词器的子词切分和上下文编码。

## 结果汇总

| 评估项 | 随机 token ids | 真实文本（探针+BPE+精炼） | 白皮书（训练分词器） |
|:-------|:---------------|:--------------------------|:---------------------|
| 八宫分类准确率 | 100.0% | 100.0% | 100.0% |
| 原型检索 Top-1 | 1.6% | **45.3%** | 71.9% |
| 原型检索 Top-5 | 7.8% | **57.8%** | — |
| 相干性均值 | 0.5896 | **0.6160** | 0.824（正确）/ 0.438（错误） |
| 相干性 > 0.7 | 0 | **12/64** | — |
| 相干性 < 0.3 | 0 | 0 | — |

**核心发现**：

1. **符号推理路径（八宫分类）不依赖文本输入**，三种条件下均为 100%。
2. **分词器质量是文本推理路径的关键瓶颈**：三阶段优化分词器将 Top-1 从 1.6% 提升至 45.3%（28 倍提升），证明模型架构的文本推理能力是有效的，之前的低分是分词器不匹配导致的假象。
3. **贪心精炼是最有效的优化手段**：纯探针映射 Top-1=34.4%，加入逐卦贪心精炼后 Top-1=45.3%（+10.9pp），Top-5 从 43.8% 提升至 57.8%（+14.0pp）。
4. **自校准机制已有效激活**：12/64 卦相干性超过 0.7，相干性分布与白皮书趋势一致。
5. **与训练分词器仍有差距**（45.3% vs 71.9%），因为探针法是逐字符独立映射，无法还原训练分词器的上下文编码和子词切分。进一步缩小差距需要开放原生分词器或其等价组件。

## 已知局限

1. **分词器为受保护资产**：仓库不包含训练时使用的分词器。`daoti_v53_tokenizer.pt` 通过探针 + BPE + 贪心精炼从模型权重中反推字符映射，是当前条件下的最优近似，但与训练分词器仍存在差距（45.3% vs 71.9%）。
2. **评估范围为 64 卦符号推理**：不涵盖 OOD 泛化、多轮对话、自然语言生成等能力——这些是后续工作的目标。
3. **无第三方基准对比**：当前无与 BERT/RoBERTa 等模型在相同任务上的对比实验。
4. **探针分词器的局限**：逐字符独立映射无法捕捉训练分词器的上下文编码特性；贪心精炼针对 64 卦文本优化，泛化到其他文本时效果可能下降。
