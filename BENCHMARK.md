# DaoTi V53 Benchmark

可复现的基准测试协议。所有评估均使用仓库中的公开模型权重和代码。

## 运行方式

```bash
python eval_benchmark.py
```

输出 `benchmark_results.json`，包含以下四项评估结果。

## 评估项目

### 1. 随机基线（Random Baseline）

8 类均匀随机猜测的期望准确率：**12.5%**

### 2. 八宫分类（Palace Classification）

对全部 64 卦执行推理，判断每卦所属八宫（乾/坤/震/巽/坎/离/艮/兑），与理法真值对比。

| 指标 | 说明 |
|:-----|:-----|
| 准确率 | 正确分类数 / 64 |
| 逐宫明细 | 每宫的正确/总数 |

**注意**：此评估使用随机 token ids 作为文本输入（分词器为受保护资产）。符号嵌入（6 位爻象 + 八卦 + 五行 + 八宫）是确定性的，因此分类准确率主要反映符号推理路径的能力。

### 3. 原型检索（Prototype Retrieval）

将文本编码后的向量与 64 个卦原型向量计算余弦相似度，检索最匹配的卦象。

| 指标 | 说明 |
|:-----|:-----|
| Top-1 准确率 | 最高相似度的卦是否正确 |
| Top-5 准确率 | 正确卦是否在相似度前 5 名 |
| 相似度统计 | 均值/最小/最大 |

### 4. 相干性分布（Coherence Distribution）

基于驻波共振腔的自校准质量信号。相干性越高，模型对自身预测越有信心。

| 指标 | 说明 |
|:-----|:-----|
| 均值/中位数 | 整体置信水平 |
| P25/P75 | 置信度分布范围 |
| 低于阈值(0.3) | 低置信度卦象数量 |
| 高于阈值(0.7) | 高置信度卦象数量 |

## 已知局限

1. **文本输入为随机 token ids**：分词器为受保护资产，无法对真实中文文本进行 tokenization。符号推理路径的结果是可靠的，文本推理路径的结果仅反映模型对随机输入的行为。
2. **评估范围为 64 卦符号推理**：不涵盖 OOD 泛化、多轮对话、自然语言生成等能力——这些是后续工作的目标。
3. **无第三方基准对比**：当前无与 BERT/RoBERTa 等模型在相同任务上的对比实验。道体模型的任务定义（64 卦结构化推理）与标准 NLP 基准不同，直接对比需设计适配的评估协议。

## 白皮书中的实验数据

以下数据来自白皮书第 5 章"能力验证"，使用完整训练管线（含分词器和真实文本）评估：

| 实验 | 指标 | 结果 |
|:-----|:-----|:-----|
| V53 合成数据八宫分类 | 准确率 | 100.00% |
| V53 纯文本自检索 | Top-1 准确率 | 71.9% |
| V60 跨域检索（白话适配器→易经原型） | Top-1 准确率 | 98.4% |
| V80 跨域同构验证（4 领域） | CKA 均值 | 0.704 |
| V34 道体 vs 随机道体语义泛化 | 检索准确率 | 30.7% vs 16.1% |
| 驻波共振自校准 | 正确/错误相干性 | 0.824 / 0.438（差距 0.386） |
| V58 多适配器生态（3 适配器） | 互不干扰 | 已验证 |

以上数据使用完整训练管线获得，与 `eval_benchmark.py` 的评估条件不同（后者使用随机 token ids），因此结果不可直接对比。
