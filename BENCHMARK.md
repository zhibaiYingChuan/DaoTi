# DaoTi V53 Benchmark

可复现的基准测试协议。所有评估均使用仓库中的公开模型权重和代码。

## 运行方式

```bash
python eval_benchmark.py
```

输出 `benchmark_results.json`，包含以下四项评估结果。

## 评估结果（使用随机 token ids）

> **评估条件**：CPU 推理，文本输入为随机 token ids（分词器为受保护资产，未公开）。
> 符号推理路径的结果是确定性的，文本推理路径的结果仅反映模型对随机输入的行为。

### 1. 随机基线（Random Baseline）

8 类均匀随机猜测的期望准确率：**12.5%**

| 指标 | 结果 |
|:-----|:-----|
| 期望准确率 | 12.5% |
| 采样准确率 | 12.5% |

### 2. 八宫分类（Palace Classification）

对全部 64 卦执行推理，判断每卦所属八宫（乾/坤/震/巽/坎/离/艮/兑），与理法真值对比。

| 指标 | 结果 |
|:-----|:-----|
| 准确率 | **100.0%** |
| 正确数 | 64 / 64 |

逐宫明细：

| 宫 | 正确/总数 | 宫 | 正确/总数 |
|:---|:----------|:---|:----------|
| 乾宫 | 8/8 | 巽宫 | 8/8 |
| 坤宫 | 8/8 | 坎宫 | 8/8 |
| 震宫 | 8/8 | 离宫 | 8/8 |
| 艮宫 | 8/8 | 兑宫 | 8/8 |

**解读**：符号嵌入（6 位爻象 + 八卦 + 五行 + 八宫）是确定性的，分类准确率反映符号推理路径的完整性。八宫分类达到 100% 说明模型的符号推理路径已完全收敛，与白皮书中 V53 合成数据八宫分类 100.00% 的结论一致。

### 3. 原型检索（Prototype Retrieval）

将文本编码后的向量与 64 个卦原型向量计算余弦相似度，检索最匹配的卦象。

| 指标 | 结果 |
|:-----|:-----|
| Top-1 准确率 | 1.6%（1/64） |
| Top-5 准确率 | 7.8%（5/64） |
| 平均相似度 | 0.6121 |
| 最小相似度 | 0.6121 |
| 最大相似度 | 0.6121 |

**解读**：检索准确率极低，且所有卦的相似度完全相同（0.6121）。原因：评估使用随机 token ids，文本编码器对相同的随机输入产生相同的特征向量，无法区分不同卦象。这与白皮书中 V53 纯文本自检索 Top-1 准确率 71.9% 形成鲜明对比——后者使用真实中文文本，文本编码器能提取语义信息进行有效检索。此结果证实：**文本推理路径依赖真实语义输入，随机输入无法激活该路径**。

### 4. 相干性分布（Coherence Distribution）

基于驻波共振腔的自校准质量信号。相干性越高，模型对自身预测越有信心。

| 指标 | 结果 |
|:-----|:-----|
| 均值 | 0.5962 |
| 中位数 | 0.5962 |
| P25 | 0.5962 |
| P75 | 0.5962 |
| 最小值 | 0.5962 |
| 最大值 | 0.5962 |
| 低于阈值(0.3) | 0 |
| 高于阈值(0.7) | 0 |

**解读**：所有卦的相干性完全相同（0.5962），集中在 0.3–0.7 的中间区间。原因同上：随机 token ids 使文本特征向量恒定，相干性（文本特征与卦原型的余弦相似度最大值）因此恒定。白皮书中驻波共振自校准的正确/错误相干性分别为 0.824 / 0.438（差距 0.386），说明在真实语义输入下相干性能有效区分预测质量。**当前结果证实：自校准机制需要真实语义输入才能发挥作用**。

## 结果汇总

| 评估项 | 随机 token ids | 白皮书（真实文本） |
|:-------|:---------------|:-------------------|
| 八宫分类准确率 | 100.0% | 100.0% |
| 原型检索 Top-1 | 1.6% | 71.9% |
| 原型检索 Top-5 | 7.8% | — |
| 相干性均值 | 0.5962 | 0.824（正确）/ 0.438（错误） |

**核心发现**：符号推理路径（八宫分类）不依赖文本输入，结果稳定；文本推理路径（检索、相干性）依赖真实语义输入，随机输入下退化为噪声。

## 已知局限

1. **文本输入为随机 token ids**：分词器为受保护资产，无法对真实中文文本进行 tokenization。符号推理路径的结果是可靠的，文本推理路径的结果仅反映模型对随机输入的行为。
2. **评估范围为 64 卦符号推理**：不涵盖 OOD 泛化、多轮对话、自然语言生成等能力——这些是后续工作的目标。
3. **无第三方基准对比**：当前无与 BERT/RoBERTa 等模型在相同任务上的对比实验。道体模型的任务定义（64 卦结构化推理）与标准 NLP 基准不同，直接对比需设计适配的评估协议。
4. **检索与相干性评估受限于输入**：随机 token ids 使文本特征恒定，导致检索准确率极低、相干性无区分度。这是评估条件的限制，不代表模型文本推理路径的能力上限。
