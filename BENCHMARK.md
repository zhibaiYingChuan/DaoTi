# DaoTi V53 Benchmark

可复现的基准测试协议。所有评估均使用仓库中的公开模型权重和代码。

## 运行方式

```bash
# 随机 token ids 模式（默认）
python eval_benchmark.py

# 真实中文文本模式（字符级分词器）
python eval_benchmark.py --real-text
```

随机模式输出 `benchmark_results.json`，真实文本模式输出 `benchmark_results_real_text.json`。

---

## A. 随机 token ids 模式

> **评估条件**：CPU 推理，文本输入为随机 token ids（分词器为受保护资产，未公开）。
> 符号推理路径的结果是确定性的，文本推理路径的结果仅反映模型对随机输入的行为。

### 1. 随机基线（Random Baseline）

8 类均匀随机猜测的期望准确率：**12.5%**

| 指标 | 结果 |
|:-----|:-----|
| 期望准确率 | 12.5% |
| 采样准确率 | 12.5% |

### 2. 八宫分类（Palace Classification）

对全部 64 卦执行推理，判断每卦所属八宫（乾/坤/震/巽/坎/离/艮/兑），与理法真值对比。

| 指标 | 结果 |
|:-----|:-----|
| 准确率 | **100.0%** |
| 正确数 | 64 / 64 |

逐宫明细：

| 宫 | 正确/总数 | 宫 | 正确/总数 |
|:---|:----------|:---|:----------|
| 乾宫 | 8/8 | 巽宫 | 8/8 |
| 坤宫 | 8/8 | 坎宫 | 8/8 |
| 震宫 | 8/8 | 离宫 | 8/8 |
| 艮宫 | 8/8 | 兑宫 | 8/8 |

**解读**：符号嵌入（6 位爻象 + 八卦 + 五行 + 八宫）是确定性的，分类准确率反映符号推理路径的完整性。八宫分类达到 100% 说明模型的符号推理路径已完全收敛，与白皮书中 V53 合成数据八宫分类 100.00% 的结论一致。

### 3. 原型检索（Prototype Retrieval）

将文本编码后的向量与 64 个卦原型向量计算余弦相似度，检索最匹配的卦象。

| 指标 | 结果 |
|:-----|:-----|
| Top-1 准确率 | 1.6%（1/64） |
| Top-5 准确率 | 7.8%（5/64） |
| 平均相似度 | 0.6121 |
| 最小相似度 | 0.6121 |
| 最大相似度 | 0.6121 |

**解读**：所有卦的相似度完全相同（0.6121）。原因：评估使用同一组随机 token ids，文本编码器对相同输入产生相同特征向量，无法区分不同卦象。此结果证实：**随机输入无法激活文本推理路径**。

### 4. 相干性分布（Coherence Distribution）

基于驻波共振腔的自校准质量信号。相干性越高，模型对自身预测越有信心。

| 指标 | 结果 |
|:-----|:-----|
| 均值 | 0.5962 |
| 中位数 | 0.5962 |
| P25 | 0.5962 |
| P75 | 0.5962 |
| 最小值 | 0.5962 |
| 最大值 | 0.5962 |
| 低于阈值(0.3) | 0 |
| 高于阈值(0.7) | 0 |

**解读**：所有卦的相干性完全相同（0.5962），无区分度。原因同上：随机输入使文本特征恒定。

## B. 真实中文文本模式（--real-text）

> **评估条件**：CPU 推理，文本输入为 64 卦各一段真实中文描述（含卦名、象义、象传原文），
> 使用字符级分词器（`ord(ch) % vocab_size`）将中文映射为 token ids。
> **注意**：字符级分词器与训练时使用的分词器不同，因此文本编码结果与训练条件存在偏差。

### 1. 八宫分类（Palace Classification）

| 指标 | 结果 |
|:-----|:-----|
| 准确率 | **100.0%** |
| 正确数 | 64 / 64 |

**解读**：与随机模式一致。八宫分类依赖符号推理路径，不受文本输入方式影响。

### 2. 原型检索（Prototype Retrieval）

| 指标 | 结果 |
|:-----|:-----|
| Top-1 准确率 | 1.6%（1/64） |
| Top-5 准确率 | 6.2%（4/64） |
| 平均相似度 | 0.4796 |
| 最小相似度 | 0.2952 |
| 最大相似度 | 0.6645 |

**解读**：检索准确率仍然很低，但相似度出现了分布（0.2952~0.6645），不再恒定。不同卦的文本描述产生了不同的文本特征向量，模型能够区分不同输入。检索准确率低的原因：字符级分词器与训练分词器的映射不一致，文本编码器无法提取正确的语义特征。白皮书中使用训练分词器的 V53 纯文本自检索 Top-1 准确率为 71.9%，说明**正确的分词器是文本推理路径的关键前提**。

### 3. 相干性分布（Coherence Distribution）

| 指标 | 结果 |
|:-----|:-----|
| 均值 | 0.4796 |
| 中位数 | 0.4646 |
| P25 | 0.4179 |
| P75 | 0.5564 |
| 最小值 | 0.2952 |
| 最大值 | 0.6645 |
| 低于阈值(0.3) | 1 |
| 高于阈值(0.7) | 0 |

**解读**：相干性出现了明显的分布差异（0.2952~0.6645，跨度 0.3693），不再恒定。1 个卦低于 0.3 阈值，说明自校准机制在真实文本输入下开始产生区分度。与白皮书中正确/错误相干性 0.824/0.438（差距 0.386）相比，当前分布的区分度仍有限，但已验证：**真实文本输入能激活自校准机制，分词器匹配度决定激活质量**。

## 结果汇总

| 评估项 | 随机 token ids | 真实中文文本（字符级分词） | 白皮书（训练分词器） |
|:-------|:---------------|:--------------------------|:---------------------|
| 八宫分类准确率 | 100.0% | 100.0% | 100.0% |
| 原型检索 Top-1 | 1.6% | 1.6% | 71.9% |
| 原型检索 Top-5 | 7.8% | 6.2% | — |
| 相干性均值 | 0.5962（恒定） | 0.4796 | 0.824（正确）/ 0.438（错误） |
| 相干性范围 | 0~0 | 0.3693 | 0.386 |
| 相干性 < 0.3 | 0 | 1 | — |
| 相干性 > 0.7 | 0 | 0 | — |

**核心发现**：

1. **符号推理路径（八宫分类）不依赖文本输入**，三种条件下均为 100%。
2. **文本推理路径（检索、相干性）依赖分词器匹配度**：随机输入→恒定输出；字符级分词→有分布但检索不准；训练分词器→检索 71.9%、相干性有效区分。
3. **自校准机制的激活需要真实语义输入**：字符级分词已使相干性产生 0.37 的分布跨度，接近白皮书中 0.386 的区分度，说明机制本身有效，瓶颈在分词器。

## 已知局限

1. **分词器为受保护资产**：仓库不包含训练时使用的分词器。字符级分词器（`ord(ch) % vocab_size`）仅用于演示文本推理路径的可激活性，其映射与训练分词器不一致，检索准确率不代表模型能力上限。
2. **评估范围为 64 卦符号推理**：不涵盖 OOD 泛化、多轮对话、自然语言生成等能力——这些是后续工作的目标。
3. **无第三方基准对比**：当前无与 BERT/RoBERTa 等模型在相同任务上的对比实验。道体模型的任务定义（64 卦结构化推理）与标准 NLP 基准不同，直接对比需设计适配的评估协议。
4. **真实文本模式的检索准确率受限于分词器偏差**：字符级分词器无法还原训练时的语义编码，导致检索准确率与白皮书数据存在显著差距。这是评估工具的限制，非模型架构的缺陷。
