# 贡献指南 | Contributing Guide

感谢您对道体基座 (DaoTi) 项目的关注！我们欢迎社区贡献，但请先阅读以下指南。

## 核心原则

**道体基座权重不开放微调。** 这是架构设计的基本决策——道体（冻结的核心编码器）是所有推理的稳定参照系，修改它将破坏退化基态结构和架构级安全机制。我们欢迎在其他模块上的贡献。

## 欢迎的贡献方向

### 1. 更高效的推理实现

- 优化 `inference.py` 中的推理速度
- 提供 ONNX / TensorRT 导出方案
- 实现 batch 推理优化

### 2. 新仿真任务的接口适配代码

- 为新领域编写适配器训练数据格式转换工具
- 实现新的物理参数适配器接口
- 贡献适配器训练的最佳实践和超参数配置

### 3. Gradio 演示改进

- 改进 `app.py` 的用户界面和交互体验
- 添加新的可视化功能（如卦象图形化展示）
- 优化移动端适配

### 4. 文档改进

- 修正文档中的错误或不准确之处
- 改进代码注释和类型标注
- 翻译文档为其他语言（英文、日文等）
- 编写更多使用示例和教程

### 5. 测试与基准

- 添加单元测试
- 在新数据集上运行基准测试并报告结果
- 发现和报告 bug

## 不接受的贡献

- **道体核心权重的微调**：道体是冻结的，不接受任何修改核心权重的 PR
- **架构源码的重建**：架构源码是受保护的商业秘密，不接受从权重反推架构的尝试
- **移除安全机制的修改**：领域分类器、相干性门控等安全机制是架构级设计，不接受绕过或移除它们的修改
- **违反许可证的用途**：不接受将模型权重用于许可证明确禁止的场景的代码

## 贡献流程

1. **Fork 仓库** 并创建您的特性分支
2. **编写代码** 确保通过现有测试
3. **提交 PR** 包含清晰的变更说明
4. **代码审查** 维护者会尽快审查

### 提交规范

```
<type>: <description>

类型:
  feat     新功能
  fix      修复 bug
  docs     文档变更
  style    代码格式（不影响功能）
  refactor 重构（不新增功能也不修复 bug）
  perf     性能优化
  test     测试相关
  chore    构建/工具变更
```

### 代码风格

- Python 3.8+ 兼容
- 遵循 PEP 8
- 函数和类添加类型标注
- 新增公共 API 需添加文档字符串

## 许可证

您提交的代码贡献将依据 **Apache 2.0** 许可证发布（与仓库代码许可证一致）。提交 PR 即表示您同意此条款。
